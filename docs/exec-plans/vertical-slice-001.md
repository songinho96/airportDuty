# Execution Plan: Vertical Slice 001

## Goal
Create the first playable slice of Airport Survival in Unity: one daytime shift with cleaning, manager scolding, stamina loss, and point gain.

## Scope
This slice does not need restaurant, check-in, information desk, thieves, ghosts, save/load, or final takeover.

## Steps
1. Pending: Create Unity project files in the repository.
2. Done: Implement `JobCategory`, `JobProgress`, `PlayerProgress`, and `ReprimandSystem` in `Assets/Scripts/Core`.
3. Done: Implement `DayResolutionService` for point gain and inactivity decay.
4. Done: Add edit-mode tests for the core rules.
5. In progress: Create `DayScene` through `Airport Survival > Build Vertical Slice 001 Scene`.
6. Done: Add a controllable player placeholder script.
7. Done: Add trash interaction and neglect timer script.
8. Done: Add manager scolding dialogue event through `AirportGameController`.
9. Done: Add HUD script for stamina, scold count, money, timer, prompt, and cleaning points.
10. Done: Add day-end summary overlay script.

## Current Implementation
- `Assets/Scripts/Gameplay/AirportGameController.cs` owns the daytime session state.
- `Assets/Scripts/Gameplay/TrashItem.cs` lets trash be cleaned and triggers manager scolding if neglected.
- `Assets/Scripts/Gameplay/SimplePlayerController.cs` and `PlayerInteractor.cs` provide placeholder movement and interaction.
- `Assets/Scripts/UI/AirportHudView.cs` displays stamina, money, scold count, timer, prompt, and manager dialogue.
- `Assets/Scripts/UI/DaySummaryView.cs` displays the day-end summary.
- `Assets/Editor/VerticalSliceSceneBuilder.cs` creates `Assets/Scenes/DayScene.unity` from the Unity menu.

## Unity Verification Steps
1. Open this folder as a Unity 3D project.
2. Wait for scripts to compile.
3. Run edit-mode tests in Unity Test Runner.
4. Click `Airport Survival > Build Vertical Slice 001 Scene`.
5. Open `Assets/Scenes/DayScene.unity`.
6. Press Play.
7. Move with WASD or arrow keys.
8. Stand near trash and press `E` to clean it.
9. Wait near uncleaned trash long enough to trigger manager scolding.
10. Confirm the HUD updates stamina, money, scold count, cleaning points, prompt, and day summary.

## Acceptance Criteria
- Player can start a day.
- Trash appears and can be cleaned.
- Ignored trash triggers manager scolding.
- Scolding increments count and reduces stamina.
- Cleaning grants cleaning points.
- Day can end and show a summary.
- Core rules are covered by edit-mode tests.

## Risks
- Unity scene state can become hard to inspect if rules are hidden in prefabs.
- Core rule tests should be implemented before adding more job minigames.
- Current scene is generated by an editor menu because no Unity project files existed when the slice scripts were added.
